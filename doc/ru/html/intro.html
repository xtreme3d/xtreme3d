Введение

GameMaker - конструктор, предназначенный в основном для создания двумерных игр и предоставляющий лишь базовые средства вывода 3D-графики. Однако, благодаря механизму расширений языка GML, можно подключить к GM внешние библиотеки, через которые реализуются практически любые недостающие функции, в том числе и высокопроизводительный рендеринг с использованием профессиональных графических API, таких, как Direct3D и OpenGL. Xtreme3D является одним из таких решений, позволяя вам создавать в GM трехмерные миры любой сложности.

Необходимо, однако, иметь в виду, что Xtreme3D не превращает GameMaker в полноценный 3D-движок наподобие Unity. Например, у вас не будет удобного 3D-редактора сцен, моделей и т.д. Все ресурсы для игры необходимо подготовить в стороннем редакторе - таком, как 3ds Max или Blender. Xtreme3D также не имеет встроенной системы хранения карт и привязки игровой логики к 3D-объектам сцены, поэтому при создании сложной игры (особенно с поддержкой модов) вам необходимо будет сделать такую систему самостоятельно.

Xtreme3D - это DLL-библиотека, функции которой вызываются хост-приложением. Взаимодействие с объектами сцены - например, изменение их свойств - осуществляется только через вызов функций. Поскольку GML поддерживает только два типа данных при вызове библиотечных функций - real и string - объекты Xtreme3D идентифицируются числами real (плавающее число двойной точности). В данном руководстве эти идентификаторы называются указателями - по аналогии с указателями из C-подобных языков. На низком уровне это действительно указатели (то есть, адреса объектов в оперативной памяти), хранящиеся как real для совместимости с GameMaker.

Xtreme3D никак не привязан к GameMaker и может быть использован в любом языке программирования, который поддерживает DLL. Для тех случаев, когда в языке нет простого способа создать окно и считывать пользовательский ввод, в движке предусмотрены соответствующие функции, так что Xtreme3D является самодостаточным решением для разработки игр - то есть, вам необязательно использовать SDL, SFML, WinAPI и т.п.

Xtreme3D основан на OpenGL 1.x и использует простейший фиксированный конвейер рендеринга, хотя и поддерживает шейдеры на GLSL. В движке есть несколько стандартных встроенных шейдеров, но любые нетривиальные эффекты и техники рендеринга вам придется реализовывать самостоятельно.
