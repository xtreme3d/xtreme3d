Xtreme3D version history

Xtreme3D 0.x-2.x (2003-2006)

The history of Xtreme3D began in 2003. The engine was written in Delphi using the GLScene library (the de facto standard 3D engine for Object Pascal) by a German programmer nicknamed Xception. Unfortunately, we do not know his real name.

The first stable version of Xtreme3D (0.9) was released in 2003. Its capabilities were rather modest: it supported only animated MD2 and MD3 models, static 3DS, MS3D and OBJ, built-in primitives, sprites, 2D and 3D text, lensflare, particles and a sky dome. The engine API resembled the style of GML functions.

The main goal of the project was to create an engine for a 3D RPG like Dungeon Master. And, although the author of the engine did not initially plan further development of Xtreme3D, in later versions the functionality of the engine quickly outgrew the needs of a specific genre.

Thus, dynamic water appeared in version 1.7 (2004). However, a much more serious breakthrough was version 2.0 (2006): a total revision of the API, support for types, material libraries and light maps, dynamic cubemapping, support for new model formats, rendering to texture, loading resources from a compressed archive, hardware shaders (bump mapping, cel shading), multitexturing, octree and quadtree, DDS support, and, most importantly, an integrated Open Dynamics Engine (ODE) physics engine.

The latest release from Xception (2.0.2.0), in addition to numerous bug fixes, contained two important innovations: a landscape renderer and a DCE collision checking engine. The engine's functionality was at the level of commercial games from 1997-2003 - Xtreme3D 2.0 allowed creating something similar in terms of graphics to Quake 2-3, Half-Life, GTA 3, The Elder Scrolls: Morrowind, etc. — everything related to the "pre-shader" era.

Unfortunately, Xception eventually abandoned its development — the engine was distributed without source code, and therefore did not develop for a long time, while technologies, meanwhile, did not stand still...

Xtreme3D 3.0 (08/25/2016)

Since 2009, the idea of ??recreating the engine from scratch, fixing bugs and adding missing functionality has been constantly floating around the Russian-speaking Xtreme3D community — considering that Xtreme3D is a very thin wrapper over GLScene, this task seemed simple and purely technical. However, unforeseen problems arose, and the matter dragged on. As a result, the project, which began in 2009 and experienced a long period of stagnation and several bursts of activity, was completed only in 2016. This is how Xtreme3D 3.0 was born.

This version of Xtreme3D, like the original, is written in Delphi using the popular GLScene library - in fact, the engine can be called almost a complete GLScene wrapper for Game Maker. Xtreme3D 3.0 is based on a modified and extended GLScene 1.0.0.0714.

Here is a short list of innovations in Xtreme3D 3.0:

  • Support for shaders in GLSL (1.1 or 1.2)
  • Dynamic shadows implemented using shadow mapping
  • 2x and 4x screen antialiasing. NVIDIA Quincunx antialiasing technology is supported
  • Fast off-screen rendering using p-buffers
  • Proxy object support
  • LOD (LODka 3D) and B3D (Blitz3D) model formats support
  • MD3 model tag support
  • Blending of skeletal animations
  • Loading materials from scripts
  • Built-in texture blending shader (TexCombine)
  • Built-in shader implementing basic Phong lighting
  • Perlin noise-based procedural textures
  • Linear wave support for dynamic water
  • New types of geometric primitives (truncated pyramid, dodecahedron, icosahedron, teapot)
  • Improved explosion effect for meshes (in particular, it is now possible to return a mesh to its original state)
  • Grid rendering
  • Debug rendering of dummycube objects
  • Many new functions for the camera
  • The ability to copy transformation matrices (local and absolute) from one object to another. You can also copy bone matrices from a model with skeletal animation to objects and set the local matrix of an object manually, which allows you to implement any mathematical model of movement and rotation of objects without being tied to Euler angles
  • The ability to apply impulses (instantaneous changes in speed) to dynamic objects in DCE. DCE also has landscape support
  • Improved ODE physics. Unlike the old implementation in Xtreme3D 2.0, ODE now fully supports Freeform objects (both static and dynamic bodies), as well as terrain. In addition, it is now possible to set positions for geometries when creating them, which allows you to create composite bodies - such as, for example, a table with legs
  • Functions for creating screenshots and saving textures from memory to BMP files
  • Function for reading text files.

Xtreme3D 3.1 (09/30/2016)

In the fall of 2016, the first update of the new Xtreme3D branch was released - version 3.1. Here is a list of the most important changes in it:

  • Improved API for Freeform objects - the ability to manually assemble Freeform from vertices and polygons, mesh transformation functions, and saving Freeform to a file
  • Support for new file formats: CSM and LMTS (which were not in 3.0), X, ASE, DXS
  • Ragdoll support in ODE
  • The Movement object, which allows you to set movement trajectories for objects
  • Improved BumpShader - support for shadow maps and automatic generation of tangent space
  • Improved PhongShader - support for textures
  • HUDShape object - 2D shape (rectangle, circle, segment and mesh are supported)
  • Improved API for sprites - support for texture atlases (i.e. using a texture fragment to draw a sprite), as well as drawing from a given point instead of the center
  • Alpha channel support for PNG
  • Functions for querying texture sizes.

Xtreme3D 3.2 (2016-10-21)

  • Support for FBOs, fast off-screen buffers that can be used to implement multi-pass rendering and various complex post-processing effects
  • New shader-compatible multitexturing mechanism for materials. Materials can now have up to 8 textures, and GLSL shaders can automatically accept them as parameters
  • ViewerRenderObject function for rendering individual objects and hierarchies
  • MaterialLoadTexture function
  • A bug in the ObjectSetParent function has been fixed.

Xtreme3D 3.3 (26.11.2016)

  • New functions for Freeform that allow reading and modifying model geometry (vertex coordinates, normals, indices, etc.)
  • Override Material support for views and FBOs. Allows you to specify a single material with which objects should be rendered, ignoring their own materials
  • Support for color buffers of different formats for FBO (including 32-bit floating point)
  • Function FBORenderObjectEx
  • Support for rendering ShadowMap shadows to a user-defined FBO instead of an internal buffer
  • Functions ViewerGetSize, ViewerGetPosition, and ViewerIsOpenGLExtensionSupported.

Xtreme3D 3.4 (12/30/2016)

  • Added support for TTF fonts and output of any Unicode characters via UTF-8 encoding
  • Added ObjectHash functions (hash table for storing any Xtreme3D objects)
  • Changed API of the FBORenderObjectEx function — new parameters have appeared that allow selectively clearing the color buffer and depth buffer, as well as copying the contents of the FBO to the main frame buffer
  • Support for passing the view and inverse view matrices to the GLSL shader parameter, as well as a special parameter that allows the shader to find out if there is a texture in a given texture block
  • In order to speed up resource loading, Freeform objects no longer generate octrees and tangent and binormal vectors when loading. This should be done manually, if necessary, using the FreeformGenTangents and FreeformBuildOctree
  • A bug in the MaterialCubeMapLoadImage function has been fixed. GLSL also now uses seamless cubemap mapping for arbitrary mipmap levels of a cubemap texture if the GL_ARB_seamless_cube_map extension is supported.

Xtreme3D 3.5 (04.02.2017)

  • Added ClipPlane object — clipping plane
  • Improved PhongShader and BumpShader shaders: added support for lsSpot light sources, fog, transparency, and rendering without lighting (if lighting is disabled in the view settings). Transparency is set via the alpha channel of the diffuse texture - or, alternatively, via the alpha value of the diffuse component of the material (Phongshader only)
  • Added new functions for materials: MaterialCullFrontFaces, MaterialSetZWrite
  • Added new ODE functions that allow you to manually set the speed, position and rotation of dynamic bodies.

Xtreme3D 3.6 (12/17/2017)

One of the largest releases in the 3.x branch — this version has been in development for over six months.

  • Added support for Windows encoding for TTF fonts, as well as custom 8-bit encodings
  • Added new functions for materials: MaterialSetTextureExFromLibrary, MaterialGetNameFromLibrary
  • Added new functions for Freeform: functions FreeformSetMaterialLibraries, FreeformMeshFaceGroupSetLightmapIndex, FreeformMeshFaceGroupGetLightmapIndex
  • Added specialized proxy objects for actors (ActorProxy)
  • Returned the functions ActorMoveBone and ActorRotateBone, added ActorMeshSetVisible
  • Returned ObjectInFrustrum
  • Returned mouse functions, added keyboard reading function
  • Added functions for creating and managing windows
  • Added functions for creating a color from RGB components
  • Added experimental functions for saving a scene to a file and reading from it
  • When creating a GLSL shader, an error message is no longer displayed if there are no errors
  • In the PhongShader and BumpShader shaders, a bug with incorrect calculation of highlights has been fixed.

Xtreme3D 3.7 (21.07.2018)

The biggest release since 3.0. This version also took more than half a year to develop.

  • A new lighting system that supports multiple light sources. Now you can create any number of sources — the 8 closest ones are taken into account when rendering an object. To do this, you need to add the LightFX effect to the object (the LightFXCreate function). Directional lights have a higher priority than point and cone lights. The system is compatible with built-in shaders and applies to all descendants of the object. The main drawback is that large objects (planes, terrain, level geometry) only take into account 8 light sources around their center, so it is recommended to use lightmapping for level lighting
  • The total number of light sources can be set using the EngineSetMaxLights function. This helps to avoid overhead with more than 8 light sources. If the value is set to 0, the engine will use the maximum number of light sources supported by the video driver. The default value is 8, but the limitation does not apply to the LightFX system.
  • Integration of the Kraft physics engine. This is a modern rigid body dynamics engine written in Delphi. It supports all basic geometries (plane, sphere, box, capsule) and static meshes, allows you to create composite geometries. Kraft is more stable than ODE, has improved collision checking and supports raycasting. You do not need an external DLL to use Kraft, since the engine is built directly into xtreme3d.dll. ODE support will be retained for the foreseeable future, but will no longer be updated
  • Functions for editing the terrain height map: BmpHDSCreateEmpty, BmpHDSSetHeight, BmpHDSGetHeight. The BmpHDSSave function saves the height map to a BMP file
  • The TerrainGetHDSPosition function returns the height map pixel position at a given point on the terrain
  • Basic support for the FBX model format. So far, it only applies to Freeform, but in the future it is planned to be added to Actor. Only the binary version of the format is supported. Materials are not loaded. The loader is based on the OpenFBX library and requires OpenFBX.dll to use it
  • New functions for proxy actors: ActorProxyObjectSetAnimationRange, ActorProxyObjectSetInterval
  • Functions for reading and unpacking PAK archives: PakGetFileCount, PakGetFileName, PakExtract, PakExtractFile. The SetPakArchive function now returns a PAK file pointer, which is required for the above functions. Compressed PAK archives are now supported
  • Added HUDSpriteGetMouseOver
  • Added ObjectGetScale
  • The ObjectDestroy function now works with all types of Xtreme3D objects
  • New functions for window management: WindowSetIcon, WindowIsShowing
  • The ViewerResetPerformanceMonitor function solves the problem with a gradual increase in FPS
  • Added output of error messages when loading resources (Freeform, Actor, textures). Messages can be disabled using the EngineShowLoadingErrors function — in this case, the engine will ignore the error and continue working.

Xtreme3D 3.7.1 (05.08.2018)

  • Functions for returning data on light sources (LightGetColor, LightGetShining, LightGetAttenuation)
  • Functions for changing and returning primitive options (SphereOptions, SphereGetOptions, CylinderOptions, CylinderGetOptions, etc.)
  • A function that returns the name of an object's material (ObjectGetMaterial)
  • Functions for returning the color and transparency of materials (MaterialGetColor, MaterialGetAlpha)
  • Functions for changing grid parameters (GridSetTille, GridSetStep).

Xtreme3D 3.7.2 (02/26/2019)

  • Function SpriteGetSize
  • Functions MaterialDestroy, MaterialSetName
  • The TextureEx system now supports 16 texture slots instead of 8 — accordingly, 16 textures can be passed to a fragment shader if the video card supports this number of texture units
  • Added the ability to unbind a shader from a material by calling MaterialSetShader with a null pointer: MaterialSetShader('material', 0);
  • Added the LineSetNode function.

Xtreme3D 3.8 (07/27/2019)

One of the largest releases in the 3.x branch — many new features, new primitives, and an integrated new physics engine for simulations of cloth, hair and soft bodies.

  • Function ViewerRenderToFilePNG — saves a screenshot in PNG format with an alpha channel
  • Function ViewerPixelRayToWorld
  • Function ViewerShadeModel — choose between flat and smooth normals
  • New type of primitive Pipe, thanks to which you can create various pipes, tentacles, etc.
  • Functions for working with Actor animation: ActorGetAnimationName, ActorGetAnimationCount, ActorAnimationDestroy, ActorAnimationNextFrame, ActorAnimationPrevFrame, ActorSetFrame
  • Function ActorTriangleCount — returns the number of triangles in the Actor
  • Functions for managing paths: MovementPathShow, MovementPathSetLoop, MovementPathDeleteNode
  • Functions for sprites: HUDSpriteXTiles, HUDSpriteYTiles
  • New TilePlane primitive for more detailed tile texturing of a plane
  • Function FireFXRingExplosion
  • Functions for working with free-form meshes: FreeformMeshObjectGetName, FreeformMeshObjectSetName, FreeformMeshObjectDestroy
  • Now the DCE manager can be detached from objects using the ObjectDestroy function and manipulated using standard functions. Example: dyn = DceDynamicSetManager(obj, mng); ObjectDestroy(dyn);
  • ObjectFindByName function — returns a pointer to an object by name
  • New VerletWorld physics engine for simulating soft objects (fabric, rope, hair, jelly) taking into account wind and gravity.

Xtreme3D 3.9 (05/20/2021)

  • Added new Engine function: EngineGetTimeStep
  • Added new Object functions: ObjectIgnoreDepthBuffer, ObjectIsPicked
  • Added new Window function: WindowDispatch
  • Invisible objects are now not taken into account when raycasts
  • Missing textures are ignored when loading a model
  • Kraft physics engine has been updated to the latest version
  • Added binding for the Lua language

Xtreme3D 3.9.1 (20.09.2021)

  • All functions now use the C calling convention (cdecl). This is done for better compatibility with various programming languages, as most languages ??have good library support for C
  • Added support for GameMaker Studio 2. For a quick setup guide, see the FAQ
  • The ViewerCreate function now checks the window handle (HWND) and throws an error if the handle is invalid or such a window does not exist.

Xtreme3D 3.9.2 (04/09/2022)

  • Added the PtrToReal function to ensure compatibility with the new version of GameMaker Studio 2
  • Added the MouseIsPressed function to read mouse button presses
  • Fixed shadow rendering when using both dynamic cube maps
  • BumpShader now uses 8 light sources and automatic tangent calculation by default (option set by BumpShaderUseAutoTangentSpace function).
